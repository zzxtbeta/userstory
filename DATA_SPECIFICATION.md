# 量子计算赛道数据采集规范

## 📋 概述

本文档定义了量子计算赛道认知系统的数据采集标准，确保数据能够支持多维度筛选和关联。

## 🎯 核心设计原则

1. **标签统一**: 所有实体（公司、人物、技术、新闻）使用统一的标签体系
2. **关联明确**: 通过 `name` 字段精确匹配实体关联
3. **细分领域**: 使用 `subdivision` 字段标注所属细分领域
4. **技术路线**: 使用 `techRoute` 标注技术路线

## 1️⃣ 公司数据格式 (quantum-companies-full.json)

### 字段说明

```json
{
  "companies": [
    {
      "id": "company-001",                    // 必填：唯一标识符
      "name": "IBM",                          // 必填：公司简称（用于关联）
      "fullName": "IBM Corporation",          // 必填：公司全称
      "location": "美国纽约",                  // 必填：总部位置
      "founded": "1911",                      // 必填：成立年份
      "valuation": "高（上市公司）",           // 必填：估值
      "stage": "成熟期",                      // 必填：发展阶段
      "description": "...",                   // 必填：公司描述
      
      // 🔑 关键字段 - 用于筛选
      "subdivision": ["量子硬件", "量子软件"],  // 必填：所属细分领域（数组）
      "techRoute": "超导量子比特",            // 必填：技术路线
      "tags": ["超导量子", "量子云服务"],      // 必填：标签（数组）
      
      "keyPeople": ["Arvind Krishna"],       // 必填：关键人物（数组，使用人物name）
      "latestRound": "已上市",                // 必填：最新融资
      "investors": ["公众股东"]               // 必填：投资方（数组）
    }
  ]
}
```

## 3️⃣ 新闻数据格式 (quantum-news-full.json)

### 字段说明

```json
{
  "news": [
    {
      "id": "news-001",                       // 必填：唯一标识符
      "title": "...",                         // 必填：新闻标题
      "date": "2024-12-15",                   // 必填：日期 (YYYY-MM-DD)
      "source": "投资界",                     // 必填：来源
      "url": "https://...",                   // 必填：原文链接
      "category": "融资动态",                 // 必填：分类
      "summary": "...",                       // 必填：内容摘要
      "aiInsight": "...",                     // 必填：AI解读
      "signalType": "利好",                   // 必填：信号类型
      "importance": 9,                        // 必填：重要性 (1-10)
      
      // 🔑 关键字段 - 用于筛选和关联
      "subdivision": ["量子硬件"],        // 必填：所属细分领域（数组）
      "techRoute": ["超导量子比特"],          // 必填：相关技术路线（数组）
      "tags": ["融资", "估值", "IPO"],        // 必填：标签（数组）
      "entities": ["本源量子", "郭国平"]      // 必填：关联实体（数组，使用精确name）
    }
  ]
}
```

### 分类标准值

- `融资动态`
- `技术动态`
- `人物动态`
- `工商动态`
- `新闻动态`

### 信号类型标准值

- `利好`
- `利空`
- `中性`

### entities 关联规则

**重要**：`entities` 数组中的名称必须与公司/人物数据中的 `name` 字段**完全一致**

示例：
```json
// 新闻中
"entities": ["IBM", "Peter Shor", "本源量子"]

// 必须对应
// 公司数据: { "name": "IBM" }
// 人物数据: { "name": "Peter Shor" }
// 公司数据: { "name": "本源量子" }
```

## ❓ 常见问题

### Q1: 如何确定一个实体属于哪个细分领域？
A: 参考 `track-overview.json` 中的定义，一个实体可以属于多个细分领域。

### Q2: 新闻的 entities 字段应该包含哪些实体？
A: 包含新闻中明确提到的公司和人物，使用它们在数据文件中的 `name` 字段。

### Q3: 如何处理一个公司有多个技术路线的情况？
A: `techRoute` 字段写主要技术路线，`tags` 字段可以包含所有相关技术。

### Q4: 标签应该写多少个？
A: 建议3-8个，覆盖技术、应用、商业等维度。
